JanuSQL

The Roman god of transitions — looking at once at past data and future growth.

A self-improving English→SQL agent on Nebius open models — it rewrites its own harness, graded by running the SQL.

Built on the SIA harness loop (SIA-H). Team SIA-for-SQL.

Making a task-specific agent better is slow.

Today, tuning an AI agent for one job is mostly manual. A person reads the failures, edits the prompt, adds a tool, fixes a parser — one change at a time. It needs an expert, and it takes a while.

  • Manual. A human reads the logs and rewrites the agent by hand.
  • Slow. Each round is read-failures → edit → re-run.
  • Expert-only. You need a person fluent in both the task and the tooling.

What if the agent did that loop on itself?

SIA: write → try → read failures → rewrite.

Three agents run a loop. One writes the agent, the target tries the task and is graded by running its SQL, and a feedback agent reads the failures and rewrites the harness for the next generation.

accuracy across generations (real)

SIA can also retrain the model’s weights (SIA-W). This build uses only the harness lever (SIA-H): the agent rewrites its own code.

Use case: English → SQL, graded by running it.

The agent reads a question in plain English and writes a SQL query. We don’t score the text — we execute the query against the database and check whether the rows it returns match the right answer. Correct means correct-when-run.

  • Input: a natural-language question.
  • Output: one SQLite query.
  • Grade: run it read-only; compare the result rows to gold.
  • Model: a fixed open model on Nebius ⓘ The model is fixed across every generation. SIA rewrites the harness around it — it adds the database schema, examples, and error-retry. The climb is a better harness unlocking the same model, not a bigger model. We pick a model strong enough to do well with a good harness, but not so strong it aces the weak first harness — or there’d be nothing to improve. This is the harness lever (SIA-H); SIA can also retrain weights (SIA-W), which this demo does not do.

            Dataset
            

              Evaluated on Spider (Yu et al., EMNLP 2018) — the
              standard text-to-SQL benchmark — its hard + extra-hard
              split: 48 questions across 6 databases. Graded by
              executing the SQL against the real database (execution
              accuracy). CC-BY-SA-4.0.
            

          

          

            Models
            

              Targets that self-improve (open, served on Nebius Token
              Factory): Gemma-3-27B-Instruct (27B dense — the
              “big” model) and NVIDIA
              Nemotron-Nano-30B-A3B (30B params, ~3B active via MoE —
              compute-light, edge-class — the “small” model).
            

            

              Harness brains (Meta + Feedback agents):
              Claude Sonnet — the agentic loop that reads the
              failures and rewrites the agent.
              Improvement classifier:
              Qwen3-30B-A3B-Instruct (structured output) on Nebius.

            What we did
            
    
              We aimed SIA at a new English→SQL task, graded by running
              the query, on open models via Nebius; and we built a research lens that
              classifies each self-improvement and flags reward-hacking.
            
    
          
    
          
    
            What we did not do
            
    
              We did not change SIA’s core algorithm. This is
              SIA-H — the agent rewrites its own code
              (the harness). It is not SIA-W: it does not retrain
              the model’s weights. We leveraged and instrumented SIA;
              we did not improve SIA itself.
            
    
          
    
          
    
            Honest replay note
            
    
              This is one real, execution-verified run, selected from several for
              clarity. The numbers are real; the playback is deterministic.
